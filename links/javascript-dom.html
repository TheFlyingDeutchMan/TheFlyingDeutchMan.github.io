To the non-tech friend of ours

Disclaimer to what I mean by non-tech:
Those that are really unfamiliar to the intricacies to technology, and are struggling to understand it.
This will be written out in laymen's terms, for easier understanding and digestion of the information given.

Javascript's relation to CSS and HTML is like the wiring to a machine, connects and powers functions in a more streamlined manner.
Where as the HTML is like the frame the wiring is built to power.
And the CSS is the design of the guts and what is on display. The coat of paint that makes it look cool.

Control flow, and loops are kinda like this:
Control flow is like following an order of steps, like following a recipe.
While loops are doing tasks repetitively until the task is done, kinda like cleaning your room till it's clean.

DOM is what makes a static webpage into something dynamic, alive even.
A way a person can interact with the DOM is for example hitting the like button on a video, this adds a +1 to the engagement number.
Or seeing a photo someone posted, and commenting on it. This creates a new box of strings that a comprised of characters you put in!
DOM is what allows pages to react live.

The difference between an array and a object is kinda like this:
An array is like a shopping list

  1. Milk
  2. Chocolate...
Now instead of counting normally, the computer starts it's count from 0. For example in the list above it'd be:
  1. 0
  2. 1
For the maths it's easy as x-1. Nothing too fancy just the way the computer counts. Important to remember.
So if you want the first item you'd request the 0th item which is the first for us.
And for objects, imagine a form. This form has information like:
  • Name
  • Age
  • Date of Birth
  • Country of Birth
  • Region
Often containing more or less depending on the type of form filled out.
To request the information, often at times you'd request the name of the person,
with Age or Date of birth as a quick security question. Though at times with a Unique Identifier
for added security.

Functions are really useful, you create a tool for your toolbox. Need to have a random number generator?
You can just make it! and if you need to use it later you can just call it again. Being in use for the task at hand!
This gives you the ability to code more dynamically in the future.
